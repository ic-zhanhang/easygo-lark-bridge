# 话题 Agent 设计：多路并行、本机存储、群话题默认读上下文

本文描述 EasyGo 飞书双 Bot（达妮娅 / 秧秧）的**目标行为**与**使用方式**。实现落在 `claw/server.ts` 补丁与 `runtime/` 规则中。

---

## 1. 设计目标

| 目标 | 做法 |
|------|------|
| 多人同时使用 | 按飞书 **话题 `thread_id`** 并行，最多 **3 个话题** 同时处理 |
| 信息不污染 | 每次只把 **当前这条 @ 消息** 交给 Agent |
| 话题可追溯 | 本机按话题 **追加存储** 用户与 Bot 消息（保留 3 天） |
| 群话题默认读历史 | 群聊话题 **默认** 附本地 jsonl 路径；私聊仍需明确要求 |
| Git | 不是每个话题都建分支；**只有要改代码时**，一个话题对应一个分支 |
| 仿真（Linux） | **整台机器** 同时只跑 **1 个** 仿真（看本机是否已有仿真在跑，与几个话题无关） |

---

## 2. 架构概览（开发向）

```
飞书群（话题模式）
  │
  ├─ 话题 A ──► Agent A
  ├─ 话题 B ──► Agent B
  └─ 话题 C ──► Agent C     （最多 3 个话题并行）

同一话题内多条 @ ──► 排队，一条接一条

每条 @ 到达
  ├─ append → runtime/topics/{thread_id}.jsonl（用户 + Bot 回复）
  └─ Agent 输入 = 当前消息（默认不含历史）

Linux 秧秧：启动仿真前检查本机
  ├─ 已有仿真在跑 → 拒绝或排队，等当前仿真结束
  └─ 无仿真在跑 → 允许启动
```

**私聊**：仅杨展航；消息同样可写入本机 `p2p-{open_id}.jsonl`。

**Mac / Linux**：各 Bot 在本机存话题文件，不跨机共享（实现细节，用户无感）。

---

## 3. 本机话题存储（开发向）

### 路径

```
runtime/topics/{thread_id}.jsonl
runtime/topics/p2p-{open_id}.jsonl   # 私聊
```

### 记录内容

用户消息与 Bot 回复均追加；字段含 `seq`、`ts`、`message_id`、`role`、`sender`、`text` 等。

### 保留

**3 天** 后清理。

### 默认 prompt

仅包含：话题 id、发送者、**当前消息正文**。不包含 jsonl 全文。

---

## 4. 读话题历史（开发向）

- **群聊话题**：每次任务默认附上 `topics/{thread_id}.jsonl` 路径，Agent 先读再执行。
- **私聊**：仅当用户明确要求读历史时才附路径（自然语言触发）。
- Agent 通过读文件拿上下文，不依赖 Claw `--resume`。
- 飞书卡片进度：群聊/私聊均显示「思考中 / 执行工具 / 回复中」。

## 5. Git 与分支（开发向）

| 情况 | 行为 |
|------|------|
| 问答、查状态、只跑命令、不改仓库 | 不建分支 |
| Agent 判断需要改代码 | 为该话题绑定 **一个** 分支，后续改代码继续用该分支 |
| 不同话题且都要改代码 | 各用各的分支；并行时用 worktree 隔离 |
| 完成 | push + MR |

分支命名建议：`feat/lark/{thread短id}-{简述}`  
绑定写入：`runtime/topics/{thread_id}.meta.json`

## 6. 并发与仿真（开发向）

| 类型 | 规则 |
|------|------|
| 普通任务 | 最多 3 个 **不同话题** 并行 |
| 同一话题 | 串行 |
| **仿真（Linux 秧秧）** | 检查 **本机当前是否已有仿真进程**（如 Gazebo / `sim_easygo.launch` / 相关容器任务），有则不再起第二个；**不是**「数有几个话题提了仿真」 |

实现要点：主机级 mutex / 进程检测，而非按话题计数。

## 7. 飞书入口（开发向）

| 入口 | 规则 |
|------|------|
| 群聊 | @Bot + 话题内消息（`thread_id`） |
| 无话题 | 可不处理（使用层已开话题模式） |
| 私聊 | 具备聊天权限的用户（见 `authorized-operator.mdc`） |
| 授权 | **仅杨展航**可授予/撤销他人聊天权限；被授权者只有聊天权限，无授权权限 |

## 8. 使用方式（Bot 第一人称，可贴群公告）

### 怎么用

1. 在群里 **开一个话题**，表示一件事。
2. 在这个话题里 **@我**，直接说你要我做什么。
3. 和这件事有关的讨论，请 **继续留在同一个话题** 里 @我。

### 我怎么处理消息

- **群聊话题**：每次 @我，我会默认结合本话题里之前的记录一起判断（存在本机，约 3 天）。
- **私聊**：默认只看当前这一条；若要结合过往，请明确说，例如：「读一下这个话题里的所有消息，然后……」。
- 答复过程中卡片会显示思考中 / 执行中等中间态。
