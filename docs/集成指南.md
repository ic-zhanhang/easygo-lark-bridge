# 集成指南：EasyGo + feishu-cursor-claw

本 Bridge 的 B 类能力：**飞书私聊 → Cursor Agent 改 EasyGo 代码 → 飞书回复**。

## 前置条件

- macOS
- [Bun](https://bun.sh)
- [Cursor Agent CLI](https://cursor.com)（`~/.local/bin/agent`）
- 飞书自建应用（本机可与 `lark-cli` 共用 `cli_a94621950d785bb3`）
- EasyGo 后端 / 前端 repo 已 clone 到 `/Users/ic/workspace/`

---

## 步骤 1：配置

```bash
cd /Users/ic/workspace/easygo-lark-bridge
cp config/easygo.env.example config/easygo.env
```

必填项：

| 变量 | 说明 |
|------|------|
| `ALLOWED_OPERATOR_OPEN_ID` | 你的 open_id（`lark-cli auth status`） |
| `FEISHU_APP_SECRET` | 飞书应用 secret |
| `CURSOR_API_KEY` | [Cursor Settings → API Keys](https://cursor.com/settings) |

可选（HEARTBEAT 用）：

| 变量 | 说明 |
|------|------|
| `GITLAB_PRIVATE_TOKEN` | 内网 GitLab API token |

---

## 步骤 2：安装

```bash
bash scripts/install.sh
```

脚本会：

1. 克隆 `feishu-cursor-claw` → `~/tools/feishu-cursor-claw`
2. 创建 `/Users/ic/workspace/easygo-claw/` 及三个 symlink
3. 部署 EasyGo `.cursor/` 模板（HEARTBEAT、rules）
4. 写入 `~/tools/projects.json`（Claw v5 要求与仓库同级）与 Claw `.env`

---

## 步骤 3：飞书开发者后台

在 https://open.feishu.cn 对应应用中：

1. 启用 **机器人**
2. 权限：`im:message`、`im:message.group_at_msg`、`im:resource`
3. 事件：`im.message.receive_v1`，**WebSocket 长连接**
4. 与 Bot **私聊**（推荐）；群聊不会执行改代码

---

## 步骤 4：验证 Agent workspace

```bash
bash scripts/verify-agent-workspace.sh
```

- 基线：`easygo-claw/` 目录必须可用
- 升级路径：探测 `easygo-dev.code-workspace` 是否可作为 `--workspace` 参数

---

## 步骤 5：试跑

```bash
cd ~/tools/feishu-cursor-claw
bun run server.ts
```

看到「飞书长连接已启动」后，**私聊** Bot：

```
修复 backend 某处 typo 并告诉我改了哪些文件
```

完成回复应包含：结论、修改文件列表、Git 状态、待确认项。

---

## 步骤 6：开机自启

```bash
bash /Users/ic/workspace/easygo-lark-bridge/scripts/claw-service.sh install
bash /Users/ic/workspace/easygo-lark-bridge/scripts/claw-service.sh status
bash /Users/ic/workspace/easygo-lark-bridge/scripts/claw-service.sh logs
```

### 运行时数据（统一在 easygo-claw/）

| 路径 | 内容 |
|------|------|
| `easygo-claw/logs/feishu-cursor.log` | Claw 服务日志 |
| `easygo-claw/inbox/` | 飞书图片/语音/文件 |
| `easygo-claw/.cursor/sessions/*.jsonl` | 对话文字记录 |
| `easygo-claw/state/sessions.json` | 会话索引 |

`install.sh` 会把 `~/tools/inbox` symlink 到 `easygo-claw/inbox/`（Claw 硬编码路径，无法改源码）。

若 HEARTBEAT 需要 GitLab token，在 launchd 环境中 export `GITLAB_PRIVATE_TOKEN`（可扩展 `claw-service.sh`）。

---

## HEARTBEAT 检查项

见 `templates/easygo-claw/.cursor/HEARTBEAT.md`：

1. 两 repo Git 状态
2. GitLab CI pipeline
3. 开放 MR

在 Claw 中通过 `/heartbeat` 或 cron 配置频率。

## 验证清单

- [ ] `bash scripts/install.sh` 成功
- [ ] `bash scripts/verify-agent-workspace.sh` 中 easygo-claw 探测通过
- [ ] 飞书**私聊** Bot → 收到进度卡片 → 收到结构化完成回复
- [ ] 非授权 open_id 或群聊消息被拒绝
- [ ] `bash scripts/claw-service.sh status` 显示 Claw 正常运行

---

## 故障排查

| 现象 | 处理 |
|------|------|
| Bot 无响应 | 检查 WebSocket 事件订阅；`bash scripts/claw-service.sh logs` |
| Agent CLI 不存在 | `cursor agent update` |
| Agent 改错目录 | 确认 `projects.json` path 为 `easygo-claw/`，非 workspace 根 |
| HEARTBEAT 无 CI | 配置 `GITLAB_PRIVATE_TOKEN` |
| 与旧自研 listener 冲突 | 勿同时运行已删除的 `bot-listener.py` |
